# Архитектура проекта Bobr

## 🎯 Обзор

Bobr — это современное одностраничное приложение (SPA), построенное на Vue 3, TypeScript и Vite. Проект представляет собой информационный сайт о бобрах с адаптивным дизайном, тёмной темой и навигацией через боковую панель.

**Основной стек:**
- **Vue 3.5** (Composition API, `<script setup>`)
- **TypeScript 5.6** (строгая типизация)
- **Vite 6.0** (сборка и dev‑сервер)
- **Vue Router 4.4** (клиентская маршрутизация)
- **CSS3** (scoped стили, CSS Grid, Flexbox, градиенты, медиа‑запросы)

## 🏗️ Архитектурные решения

### 1. **Компонентная архитектура**
- **App.vue** — корневой компонент, содержит `SideBar` и `RouterView`.
- **SideBar.vue** — единственный переиспользуемый компонент, отвечает за навигацию и адаптивное меню.
- **Views** — страницы, соответствующие маршрутам. Каждый view независим и содержит свою логику и стили.

### 2. **Состояние (State Management)**
Централизованное состояние (Pinia/Vuex) не используется, так как приложение статическое и не требует сложного управления данными.

**Локальное состояние:**
- `ref` / `reactive` внутри компонентов (например, `isOpen` в `SideBar.vue` для управления мобильным меню).
- Массивы данных (галерея, факты) объявлены непосредственно в компонентах.

**Преимущества:** простота, отсутствие накладных расходов.
**Недостатки:** при масштабировании может потребоваться централизованный store.

### 3. **Маршрутизация (Vue Router 4)**
Конфигурация в `src/router/index.ts`:
- Используется `createRouter` с `createWebHistory` (HTML5 History Mode).
- Все маршруты загружаются синхронно (ленивая загрузка не требуется из‑за небольшого размера).
- Каждый маршрут сопоставлен с соответствующим view‑компонентом.

```typescript
const routes: RouteRecordRaw[] = [
  { path: '/', name: 'home', component: HomeView },
  { path: '/bobr', name: 'bobr', component: BobrView },
  // ...
]
```

### 4. **Стили и дизайн**
- **Глобальные стили** (`style.css`) — сброс отступов, скроллбар, базовые настройки.
- **Scoped стили** в каждом компоненте — изоляция, предотвращение конфликтов.
- **Адаптивность:** mobile‑first, медиа‑запросы в каждом компоненте.
- **Дизайн‑система:**
  - Цветовая палитра: тёмный фон (`#0f0f1a`), золотые градиенты (`#8B6914` → `#D4A030`).
  - Типографика: системные шрифты, градиентный текст для заголовков.
  - Эффекты: backdrop‑blur, плавные переходы, hover‑анимации.

### 5. **Обработка ассетов**
- **Изображения:** импортируются напрямую в компоненты через `import` (TypeScript).
- **SVG:** используются как `img` или встроенные иконки через эмодзи‑фолбэк.
- **Оптимизация:** Vite автоматически обрабатывает ассеты (хэширование, сжатие).

### 6. **TypeScript**
- Строгий режим (`strict: true`).
- Алиасы путей (`@/` → `./src/`).
- Типизация пропсов и событий не требуется (пропсы не используются).
- Типизация данных в компонентах (массивы объектов).

## 🔧 Паттерны и практики

### **Composition API (`<script setup>`)**
Все компоненты используют синтаксис `<script setup lang="ts">`, что обеспечивает:
- Локальную реактивность через `ref`, `reactive`.
- Прямое использование хуков жизненного цикла (`onMounted`, `watch`).
- Чистую логику без boilerplate.

### **Реактивность**
- Простая реактивность через `ref` (например, `isOpen` в SideBar).
- Отсутствие сложных вычисляемых свойств (computed) и watchers (кроме отслеживания маршрута).

### **Адаптивная навигация**
SideBar реализует паттерн **Conditional Rendering**:
- На десктопе — фиксированная панель.
- На мобильных — гамбургер‑меню с оверлеем и анимацией.

### **Композиция стилей**
- **Utility‑классы** не используются (нет Tailwind).
- **CSS‑переменные** не используются, цвета заданы явно.
- **БЭМ**‑подобное именование классов (`.sidebar`, `.sidebar-brand`).

### **Импорт ассетов**
Паттерн **Direct Import**:
```typescript
import damBuilder from '@/assets/gallery/dam-builder.jpg'
```
Vite преобразует импорты в оптимизированные URL.

## 🚀 Сборка и развёртывание

### **Vite Configuration** (`vite.config.ts`)
- Плагин `@vitejs/plugin-vue` для обработки `.vue` файлов.
- Алиас `@` для удобных импортов.
- Dev‑сервер с горячей перезагрузкой.

### **Скрипты npm**
- `npm run dev` — запуск dev‑сервера на `localhost:5173`.
- `npm run build` — production‑сборка в `dist/`.
- `npm run preview` — локальный просмотр собранного приложения.

### **Production‑сборка**
- Vue‑компоненты компилируются в оптимизированный JavaScript.
- CSS минифицируется и извлекается в отдельные файлы.
- Ассеты хэшируются для кэширования.

## 📊 Оценка архитектуры

### ✅ **Сильные стороны**
1. **Простота и ясность** — минимальная сложность, идеально для статического сайта.
2. **Современный стек** — Vue 3, Composition API, TypeScript.
3. **Полная адаптивность** — мобильный и десктопный UX.
4. **Чистая структура** — разделение на views/components/router.
5. **Быстрая сборка** — Vite обеспечивает мгновенный hot‑reload.

### ⚠️ **Области для улучшения**
1. **Отсутствие тестов** — нет unit‑ или e2e‑тестов.
2. **Нет линтинга/форматирования** — ESLint, Prettier не настроены.
3. **Дублирование стилей** — медиа‑запросы повторяются в каждом компоненте.
4. **Нет централизованного состояния** — при добавлении динамики (например, избранное) потребуется Pinia.
5. **Нет SEO‑оптимизации** — статические мета‑теги, отсутствие SSR.

### 🔮 **Рекомендации по развитию**
1. **Добавить тесты** — Vitest + Vue Test Utils для компонентов.
2. **Настроить линтинг** — ESLint с `@vue/typescript`, Prettier.
3. **Вынести общие стили** — CSS‑переменные для цветов, миксины для медиа‑запросов.
4. **Внедрить Pinia** — для будущего состояния (например, избранные изображения).
5. **Добавить SSR/SSG** — VitePress или Nuxt для SEO.
6. **Оптимизировать изображения** — конвертация в WebP, lazy loading.

## 🧩 Заключение

Архитектура Bobr следует современным best practices для небольших Vue‑приложений. Проект хорошо структурирован, использует Composition API и TypeScript, полностью адаптивен. Для текущего scope (статический сайт) архитектура избыточна, что является плюсом для поддержки и масштабирования.

**Ключевые паттерны:**
- **Composition over Options API** — логика внутри `<script setup>`.
- **View‑Component Separation** — чёткое разделение страниц и переиспользуемых компонентов.
- **Mobile‑First CSS** — стили начинаются с мобильных устройств.
- **Direct Asset Import** — типобезопасный импорт изображений.

Проект готов к расширению: добавлению API‑запросов, глобального состояния, тестов и CI/CD.
