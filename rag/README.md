# RAG База проекта Bobr

Retrieval-Augmented Generation (RAG) система для семантического поиска по коду и документации проекта.

## Структура

- `index.json` — основной индекс чанков (разбитые на части файлы).
- `embeddings.json` — эмбеддинги векторов (пока не генерируются).
- `create_index.cjs` — скрипт для создания/обновления индекса.
- `query.cjs` — интерактивный поиск по индексу.
- `create_embeddings.cjs` — скрипт для генерации эмбеддингов (требует Ollama).

## Использование

### 1. Создание индекса
```bash
node rag/create_index.cjs
```
Скрипт просканирует все текстовые файлы проекта (исключая бинарные) и создаст `index.json`.

### 2. Поиск по ключевым словам
```bash
node rag/query.cjs "ваш запрос"
```
Или интерактивный режим:
```bash
node rag/query.cjs
```

Примеры:
```bash
node rag/query.cjs sidebar
node rag/query.cjs "vue router"
node rag/query.cjs "адаптивный дизайн"
```

### 3. Генерация эмбеддингов (опционально)
Для семантического поиска нужны векторные представления. Если у вас есть локальный Ollama с моделью эмбеддингов (например, `nomic-embed-text`), запустите:
```bash
node rag/create_embeddings.cjs
```
Скрипт создаст `embeddings.json` с векторами для каждого чанка.

## Как работает индексация

1. **Фильтрация файлов** — исключаются `node_modules`, `.git`, изображения, логи.
2. **Разбиение на чанки** — каждый файл делится на перекрывающиеся блоки по 20 строк с перекрытием 5 строк.
3. **Метаданные** — для каждого чанка сохраняются:
   - исходный файл
   - язык (vue, typescript, css, ...)
   - диапазон строк
   - текст чанка
   - приблизительное количество токенов

## Поиск

### Keyword Search
Простой поиск по вхождению слов (без морфологии, регистронезависимый). Подсчитывает количество вхождений каждого слова запроса в чанке и ранжирует по убыванию.

### Semantic Search (в будущем)
Планируется использовать эмбеддинги Ollama для векторного поиска. Запрос будет преобразовываться в вектор, затем искать ближайшие чанки по косинусному сходству.

## Расширение

### Добавление новых типов файлов
Отредактируйте `create_index.cjs`, добавив расширение в `switch` (определение языка) и при необходимости обновите фильтр исключений.

### Изменение размера чанка
Измените константы `chunkSize` и `overlap` в скрипте индексации.

### Интеграция с LLM
Можно использовать найденные чанки как контекст для генерации ответов с помощью OpenAI, Claude или локальных моделей через Ollama.

## Примечания

- Индекс не обновляется автоматически при изменении файлов. Перезапускайте `create_index.cjs` после значительных правок.
- Эмбеддинги требуют модель с поддержкой эмбеддингов в Ollama. Если её нет, semantic search недоступен.
- Для больших проектов考虑 использовать специализированные инструменты (например, `chromadb`, `qdrant`).

## Пример использования в CI/CD

```bash
# Перед сборкой обновить индекс
node rag/create_index.cjs

# Проверить, что документация содержит ключевые термины
node rag/query.cjs "LICENSE" | grep -q "MIT" || echo "Лицензия не найдена"
```
